# Lessons (Project-Lynx build)

- **blockscout `read_contract` coerces uint256 → JS float** (scientific notation, precision lost). For exact 256-bit ids, cross-check against a known source (Gamma clobTokenIds) or capture exact values via Tenderly `get_vnet_simulation_asset_changes` / a raw eth_call. Don't paste a float into code.
- **OpenAI and Anthropic are separate NegRisk markets**, each with its own negRiskMarketID. The docs implied one shared "AI-model" market — verify market grouping from Gamma's `negRiskMarketID` per market, not assumptions.
- **Gamma `questionID` (capital ID)** is the field that drives `getPositionId`. `outcomes`/`outcomePrices`/`clobTokenIds` are stringified JSON arrays — `JSON.parse` before indexing.
- **Never weaken a failing test to make it green** (the false-green USDC-balance trap). The EnterBasket fork test must assert recipient holds BOTH outcome tokens AND the contract retains zero wcol/USDC.e/tokens.
- **NegRiskAdapter.splitPosition `safeBatchTransferFrom`s the set to msg.sender** → an executor contract MUST implement `onERC1155BatchReceived` (inherit OZ `ERC1155Holder`) or the split reverts. The plan omitted this; the fork test caught it. Always fork-test a contract that *receives* ERC-1155 from a real protocol.
- **EnterBasket fund-stranding via mismatched (conditionId, questionId):** `splitPosition(conditionId)` mints a real set but `getPositionId(questionId)` reads ids the contract never received → silent strand. Guard with `require(adapter.getConditionId(questionId) == conditionId)` up front. (Found by the 2-engine review.)
- **Client/server secret boundary in Next.js:** a module that throws at import on a missing server-only env (config.ts) must NOT be transitively imported by any `"use client"` component, or it crashes the browser bundle. Put public constants (addresses) in a separate client-safe module.
- **Concurrent git process on the same repo:** a parallel agent committed ("v1 stack", "assets") and ran `git checkout main` mid-build, stranding me on main with uncommitted work. Recovery: commits are safe on the feature branch ref; `git reflog` + `git checkout -f build/lynx-mvp` restored state. Lesson: commit small + often, re-verify `git branch --show-current` after each commit, and read COMMITTED blobs (`git show branch:path`) when the working tree is volatile.
- **Verify API shapes against current docs/SDKs, not assumptions:** docs/05 claimed Uniswap `/quote` `q.quote` is a string — it's actually an OBJECT (`quote.output.amount`, `quote.swapper`). A parallel research workflow (context7 + WebFetch + MCP) caught it before it shipped a broken oracle.
